# 02. 快速开始

本篇带你从环境安装到完成第一次预测。

## 2.1 环境安装

要求 Python 3.10 及以上版本：

```bash
pip install -r requirements.txt
```

## 2.2 使用 KronosPredictor 进行预测

`KronosPredictor` 类封装了**数据预处理、归一化、预测、反归一化**全流程，几行代码即可从原始数据得到预测结果。

### 步骤 1：加载 Tokenizer 与模型

从 Hugging Face Hub 加载预训练的 Kronos 模型及其对应的 Tokenizer：

```python
from model import Kronos, KronosTokenizer, KronosPredictor

# 从 Hugging Face Hub 加载
tokenizer = KronosTokenizer.from_pretrained("NeoQuasar/Kronos-Tokenizer-base")
model = Kronos.from_pretrained("NeoQuasar/Kronos-small")
```

### 步骤 2：实例化 Predictor

传入模型、Tokenizer 与目标设备：

```python
# 初始化预测器；device 不传时会自动选择 cuda / mps / cpu
predictor = KronosPredictor(model, tokenizer, max_context=512)
```

### 步骤 3：准备输入数据

`predict` 方法需要三个主要输入：

- `df`：包含历史 K 线数据的 pandas DataFrame，必须包含 `['open', 'high', 'low', 'close']` 列；`volume`、`amount` 可选。
- `x_timestamp`：与 `df` 历史数据对应的时间戳 Series。
- `y_timestamp`：希望预测的未来各时间点的时间戳 Series。

```python
import pandas as pd

# 加载数据
df = pd.read_csv("./data/XSHG_5min_600977.csv")
df['timestamps'] = pd.to_datetime(df['timestamps'])

# 定义上下文窗口与预测长度
lookback = 400   # 用于输入的历史步数
pred_len = 120   # 需要预测的未来步数

# 为预测器准备输入
x_df = df.loc[:lookback-1, ['open', 'high', 'low', 'close', 'volume', 'amount']]
x_timestamp = df.loc[:lookback-1, 'timestamps']
y_timestamp = df.loc[lookback:lookback+pred_len-1, 'timestamps']
```

### 步骤 4：生成预测

通过 `T`、`top_p`、`sample_count` 等参数控制采样过程：

```python
pred_df = predictor.predict(
    df=x_df,
    x_timestamp=x_timestamp,
    y_timestamp=y_timestamp,
    pred_len=pred_len,
    T=1.0,          # 采样温度（越大越随机）
    top_p=0.9,      # 核采样（nucleus sampling）概率阈值
    sample_count=1  # 生成并平均的预测路径数量
)

print("预测结果前几行：")
print(pred_df.head())
```

`predict` 返回一个 pandas DataFrame，包含 `open`、`high`、`low`、`close`、`volume`、`amount` 的预测值，索引为传入的 `y_timestamp`。

## 2.3 批量预测

当需要同时对多个标的或多个时间段预测时，使用 `predict_batch` 可并行处理多组数据：

```python
# 准备多组数据
df_list = [df1, df2, df3]
x_timestamp_list = [x_ts1, x_ts2, x_ts3]
y_timestamp_list = [y_ts1, y_ts2, y_ts3]

# 批量预测
pred_df_list = predictor.predict_batch(
    df_list=df_list,
    x_timestamp_list=x_timestamp_list,
    y_timestamp_list=y_timestamp_list,
    pred_len=pred_len,
    T=1.0,
    top_p=0.9,
    sample_count=1,
    verbose=True
)

for i, pred_df in enumerate(pred_df_list):
    print(f"序列 {i} 的预测：")
    print(pred_df.head())
```

### 批量预测的重要约束

- 所有序列必须具有**相同的历史长度（lookback 窗口）**。
- 所有序列必须具有**相同的预测长度（`pred_len`）**。
- 每个 DataFrame 必须包含 `['open', 'high', 'low', 'close']` 列。
- `volume`、`amount` 列可选，缺失时会用 0 填充。

## 2.4 关键采样参数说明

| 参数 | 含义 | 建议 |
| --- | --- | --- |
| `T`（temperature，温度） | 控制采样随机性。值越大分布越平滑、越随机；越小越确定。 | 常用 0.6 ~ 1.0 |
| `top_k` | 仅保留概率最高的 k 个 token 进行采样。0 表示不启用。 | 0（默认关闭） |
| `top_p`（nucleus） | 仅保留累计概率达到 p 的最小 token 集合。 | 0.9 左右 |
| `sample_count` | 生成多少条预测路径，最终取均值以降低方差。 | 1 ~ 5 |

> 提示：增大 `sample_count` 可获得更平滑稳定的预测，但推理耗时线性增加。
