# 05. 微调指南

Kronos 提供两套微调流程，分别面向不同数据源：

- **`finetune/`**：基于 **Qlib** 量化数据库的微调流程，适合 A 股全市场、配套回测。
- **`finetune_csv/`**：基于**自定义 CSV 文件**的轻量微调流程，适合单标的或自有数据。

两套流程都遵循「先微调 Tokenizer，再微调 Predictor（主模型）」的顺序。

---

## 5.1 基于 Qlib 的微调（finetune/）

### 配置

全局配置集中在 [finetune/config.py](../finetune/config.py) 的 `Config` 类，关键项：

| 配置项 | 含义 |
| --- | --- |
| `qlib_data_path` | Qlib 数据目录（需自行准备）。 |
| `instrument` | 股票池，如 `csi300`。 |
| `lookback_window` | 输入历史步数（默认 90）。 |
| `predict_window` | 预测未来步数（默认 10）。 |
| `max_context` | 模型最大上下文（512）。 |
| `feature_list` | 使用特征：`['open','high','low','close','vol','amt']`。 |
| `train/val/test_time_range` | 数据集时间切分；注意验证/测试集起点会前移以容纳 `lookback_window`。 |
| `epochs`, `batch_size`, `*_learning_rate` | 训练超参数。 |
| `pretrained_tokenizer_path` / `pretrained_predictor_path` | 预训练权重路径（本地或 HF）。 |
| `use_comet`, `comet_config` | Comet ML 实验追踪配置（可选）。 |
| `backtest_*` / `inference_*` | 回测与推理采样参数。 |

### 完整流程

```mermaid
flowchart LR
    A[Qlib 原始数据] --> B[qlib_data_preprocess.py 预处理]
    B --> C[处理后的 pkl 数据集]
    C --> D[train_tokenizer.py 微调 Tokenizer]
    D --> E[train_predictor.py 微调 Predictor]
    E --> F[qlib_test.py 回测评估]
```

1. **数据预处理**：运行 [finetune/qlib_data_preprocess.py](../finetune/qlib_data_preprocess.py)，从 Qlib 加载并按时间切分，序列化为 `train_data.pkl` / `val_data.pkl`。
2. **数据集封装**：[finetune/dataset.py](../finetune/dataset.py) 的 `QlibDataset` 预计算所有滑动窗口起点，按 epoch 随机采样；`window = lookback_window + predict_window + 1`，并即时生成时间特征。
3. **微调 Tokenizer**：[finetune/train_tokenizer.py](../finetune/train_tokenizer.py)，基于 DDP 分布式训练。
4. **微调 Predictor**：[finetune/train_predictor.py](../finetune/train_predictor.py)，同样支持 DDP；`create_dataloaders` 构造分布式 DataLoader。
5. **回测评估**：[finetune/qlib_test.py](../finetune/qlib_test.py)。

### 分布式启动示例

```bash
# 单机多卡（torchrun）
torchrun --standalone --nproc_per_node=NUM_GPUS train_tokenizer.py
torchrun --standalone --nproc_per_node=NUM_GPUS train_predictor.py
```

### 通用训练工具

[finetune/utils/training_utils.py](../finetune/utils/training_utils.py) 提供：

- `setup_ddp` / `cleanup_ddp`：初始化/销毁分布式进程组。
- `set_seed`：设置随机种子，保证可复现。
- `get_model_size`：统计模型参数量。
- `format_time`：格式化耗时显示。

## 5.2 基于 CSV 的微调（finetune_csv/）

适合使用自有 CSV 数据快速微调。详细中文说明见 [finetune_csv/README_CN.md](../finetune_csv/README_CN.md)。

### 数据格式

CSV 必须包含以下列：`timestamps, open, high, low, close, volume, amount`。
其中 `volume` 与 `amount` 若无数据可全部填 0。标准样例：[finetune_csv/data/HK_ali_09988_kline_5min_all.csv](../finetune_csv/data/HK_ali_09988_kline_5min_all.csv)。

### 配置文件（YAML）

配置示例：[finetune_csv/configs/config_ali09988_candle-5min.yaml](../finetune_csv/configs/config_ali09988_candle-5min.yaml)。需修改 `data_path` 与预训练模型路径：

```yaml
data:
  data_path: "/path/to/your/data.csv"
  lookback_window: 512   # 使用的历史数据点
  predict_window: 48     # 预测的未来点数
  max_context: 512       # 最大上下文长度
```

配置由 [finetune_csv/config_loader.py](../finetune_csv/config_loader.py) 的 `CustomFinetuneConfig` 加载。

### 训练方式

**方式一：顺序训练（推荐）** — [finetune_csv/train_sequential.py](../finetune_csv/train_sequential.py) 自动完成「Tokenizer → Predictor」全流程：

```bash
# 完整训练
python train_sequential.py --config configs/config_ali09988_candle-5min.yaml

# 跳过已存在模型
python train_sequential.py --config configs/config_ali09988_candle-5min.yaml --skip-existing

# 只训练 tokenizer
python train_sequential.py --config configs/config_ali09988_candle-5min.yaml --skip-basemodel

# 只训练 predictor
python train_sequential.py --config configs/config_ali09988_candle-5min.yaml --skip-tokenizer
```

**方式二：分组件训练**：

```bash
# 步骤 1：训练 tokenizer
python finetune_tokenizer.py --config configs/config_ali09988_candle-5min.yaml
# 步骤 2：训练 predictor（依赖微调后的 tokenizer）
python finetune_base_model.py --config configs/config_ali09988_candle-5min.yaml
```

`SequentialTrainer` 会自动读取环境变量 `RANK` / `WORLD_SIZE` / `LOCAL_RANK` 以支持 DDP，并在 `world_size > 1` 且有 CUDA 时初始化进程组。

## 5.3 微调建议

- **先 Tokenizer 后 Predictor**：Predictor 依赖 Tokenizer 产出的离散 token，务必保持顺序。
- **学习率**：Tokenizer 与 Predictor 通常使用不同学习率（见各自配置）。
- **上下文一致性**：`max_context`、`s1_bits`、`s2_bits` 必须与预训练权重保持一致。
- **数据质量**：确保无 NaN，价格与成交量列完整；归一化在数据集/预测器内部完成。
